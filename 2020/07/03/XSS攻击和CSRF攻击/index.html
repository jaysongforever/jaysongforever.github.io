xss攻击是什么

xss叫跨站脚本攻击，Cross Site Scripting，WEB安全领域为了和css区分开来，于是改成xss。xss大致分为三种，反射型，存储型，DOM型。

1.反射型例子：用户点击了某个链接发送一个请求到攻击者起的一个服务器，攻击者从服务端响应回去一段script包裹的恶意代码，浏览器拿到代码后解析为js代码，执行某些操作。
2.存储型例子：用户输入某些用script包裹的代码文本传给服务器，服务器不做处理的又将这些文本传给其他用户，浏览器拿到代码后解析为js代码，执行某些操作，类似于用户通过富文本输入，用$.html和innerHTML解析代码容易出现此类攻击

CSRF攻击是什么

CSRF全称叫Cross Site Request Forgery(跨站资源伪造),可以利用Cookie发起CSRF攻击。
用户Cookie有时候保存了用户的一些信息，刚登录的用户，后台返回一个带有用户信息的页面，每次访问服务器，请求都会携带这个Cookie去服务器，服务器可以由此来识别用户身份。登录的用户会发请求去删除他的某个东西，请求如下

预防XSS攻击

  • HttpOnly 防止劫取 Cookie，设置了httponly后，浏览器精致js代码无法获取设置了httponly的Cookie
  • 输入检查，前端进行输入过滤,不能将用户输入的东西作为可信来源，将<>之类的进行转义&lt;<,&gt;>,导入xss插件，做标签白名单过滤
  • 输出检查，后端在将内容返回给前端的时候进行过滤

预防CSRF攻击

  • 设置get请求不能对数据进行修改
  • 限制第三方网站访问到用户Cookie
  • 阻止第三方网站请求接口
  • 请求时附带验证信息，比如验证码或者token
